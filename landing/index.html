Syncfox

Self-hosted multi-cloud bidirectional sync hub.
rclone bisync + a friendly operator UI.

Quickstart → GitHub Docker Hub

What it does

Connect two cloud accounts (Google Drive, iCloud Drive, Dropbox), pick a folder on each, and Syncfox keeps them in lockstep. Change-detection on both sides, rclone bisync as the engine, and a UI that warns you 25 days before iCloud's trust token dies — so syncs don't break at 3 AM.

Why

rclone bisync is excellent and miserable to operate. You have to write the right config stanza per provider, redo iCloud's 2FA dance every ~30 days, script polling so changes propagate in seconds, and not lose credentials when you bounce the host. Syncfox is the operator UI that handles those four things. The actual sync is still rclone — fork it, replace it, audit it.

Features

3 providers v1Google Drive (OAuth + service-account), iCloud Drive (Apple ID + 2FA), Dropbox (PKCE OAuth).
Per-pair watchersDrive changes.list watermark + iCloud root-folder fingerprint. ~30s end-to-end propagation.
iCloud expiry surfaceGreen/orange/red badge per remote. One-click re-auth. Optional daily Discord nudge ≤25d.
Onboarding wizard/setup walks first-run users through provider-1 → provider-2 → first pair, then steps aside.
Opt-in encryptionSet SYNCFOX_MASTER_PASSWORD → Fernet-encrypted creds at rest (PBKDF2-SHA256, 200k).
Multi-arch imagelinux/amd64 + linux/arm64. Pi 4/5, Apple Silicon, x86 server — all the same image.
No telemetryNo phone-home, no signup, no analytics. Your container, your data, your network.
MIT licensedFork it, replace it, audit it. No paid tier coming. No license-key gate.

Quickstart

mkdir syncfox && cd syncfox
curl -O https://raw.githubusercontent.com/sahirmathur1/syncfox/main/docker-compose.yml
curl -o .env https://raw.githubusercontent.com/sahirmathur1/syncfox/main/.env.example
# edit .env — only PUBLIC_BASE_URL is required for local-only
docker compose up -d
open http://localhost:8081

The container initialises SQLite, starts the web UI on port 8081, and redirects to /setup until you've connected two accounts and made one pair.

Stack

Python 3.12 / FastAPI / uvicorn / SQLite (WAL) / rclone bisync / cryptography for opt-in Fernet encryption. Server-rendered HTML, no JS framework, no build step. Tests under tests/unit/, run uv run pytest -q.
